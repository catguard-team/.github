# 자경단 문서 스타일 가이드

> 자경단 문서는 정중하고 솔직한 톤으로, GitHub에서 보기에도 단정하게 정리되어 있어야 합니다.
> 이 가이드는 그 "단정함"의 자경단스러운 정의입니다.

---

## 톤 (제일 중요한 한 줄)

자경단 문서의 톤은 **이야기 중심 정중체**를 기준으로 합니다. 인사로 시작, 솔직한 회상, 구체적 사례, 일상 비유, 자문자답, 부드러운 결론 유도. 이 결을 모든 문서에 일관되게 적용합니다.

### 권장하는 표현

- "안녕하세요. ~입니다." (인사로 시작)
- "~인데요", "~이고요", "~잖아요" (대화체 연결)
- "여러분들이", "단원분들이", "새끼고양이분들이" (직접 호명, 정중)
- "저는 ~라고 생각합니다" (의견을 부드럽게)
- "~하시면 됩니다", "~해 주세요" (안내·요청)
- "~지 못합니다", "~기 어렵다고 봅니다" (한계 인정)
- "고맙습니다." (마무리)

### 피하는 표현

- "안 받습니다 / 안 해줍니다 / 끝." 같은 단정·단절 어미
- "고작 ~", "솜방망이", "독고다이" 같은 자기비하적이지만 공격적인 자조
- "괜히 멋있게 쓴 척" 같은 메타-자조
- "❌ X 안 함 / Y 안 함" 식 부정 나열만으로 정체성을 만드는 방식 (필요하면 사용 가능, 단 보조적으로)
- 시민단체 어휘: "함께 만들어가는", "마음을 모아", "따뜻한 세상", "헌신적인", "위대한"

### 자경단 고유 표현 (유지)

브랜드 정체성이라 그대로 둡니다.

- **냐-옹.** (사인오프, 정중체와 함께: "고맙습니다. 냐-옹.")
- **새끼고양이 / 노묘 / 자경단원 / 길고양이** (구성원 호칭)
- **꿀잠 스팟 / 첫 쥐돌이 / 그루밍** (활동 메타포)
- **귀찮지만 한다** (내부용 슬로건 — 외부 문서에서는 "작은 일을 가까이서 합니다"로 풀어 써도 됩니다)

## GitHub Callout (적극 사용)

GitHub은 5종 callout을 지원합니다. 자경단 톤에 맞춰 이렇게 씁니다.

```markdown
> [!NOTE]
> 알아두시면 좋은 컨텍스트입니다.

> [!TIP]
> 자경단원이 발견한 더 나은 방법입니다.

> [!IMPORTANT]
> 빠뜨리시면 안 되는 핵심입니다.

> [!WARNING]
> 잘못하시면 새끼고양이가 다칠 수 있습니다.

> [!CAUTION]
> 절대 하지 말아주세요. 사고 직전입니다.
```

> [!CAUTION]은 SAFETY 위반·데이터 누출 같은 **돌이킬 수 없는** 케이스에만 사용합니다.

## 표 (Table)

- 헤더 행은 항상 있어야 합니다.
- 컬럼 정렬: 텍스트 좌측, 숫자 우측 (`---:`).
- 컬럼 수는 4개 이하를 권장합니다. 더 필요하시면 표를 분할해 주세요.

```markdown
| 항목 | 무엇 | 단가 |
|------|------|----:|
| 케이스 | 정품 팬 포함 | 18,000 |
```

## 이모지 (제한적)

| 영역 | 사용 OK | 사용 ❌ |
|------|---------|---------|
| 카테고리 라벨 | 🛠️ 🐙 🐍 🇰🇷 🍎 🐧 🪟 🌐 📚 | 💖 🌟 🎉 🌈 🦄 |
| 자경단 메타포 | 🐾 🐭 🍯 🛡️ 📡 | (대부분 OK) |
| 상태 | ✅ ❌ ⚠️ 🚨 | 🟢🔴🟡 점 (대신 텍스트 사용) |
| 사용하지 않음 | — | 🌍 (지구본), ❤️ (하트), 👏 (박수), 🤝 (악수), 👑 (왕관) |

> 한 README에 이모지 5개 이하를 권장합니다. 카테고리 표는 예외입니다.

## 헤더 위계

- `#` 레포 이름 (한 번만, 보통 hero 영역에서 사용)
- `##` 주요 섹션 (3~7개 권장)
- `###` 하위 섹션
- `####` 거의 안 씁니다 (필요하시면 표로 대체)

> 헤더에 이모지 하나 정도는 괜찮습니다. 두 개 이상은 자경단 결과 어울리지 않습니다.

## 코드 블록

언어 태그를 항상 명시해 주세요.

````
```bash
# 명령
```
```python
# 코드
```
```yaml
key: value
```
````

코드 안에 비밀 정보 예시를 넣으실 때는 `<TOKEN>` 같은 placeholder로 표기해 주세요. 진짜처럼 보이는 가짜도 사용하지 않습니다 ([LEAK_CLEANUP §8](https://github.com/catguard-team/manifesto/blob/main/LEAK_CLEANUP.md)).

## 링크

- 같은 레포 내: 상대 경로 `[X](./PATH.md)`
- 다른 자경단 레포: 절대 경로 `https://github.com/catguard-team/<repo>/...`
- 외부: 신뢰 가능한 출처로 한정합니다 (`.go.kr`, 공식 문서, GitHub, 검증된 오픈소스).

## Sign-off

모든 README, 매니페스토 문서는 다음으로 끝맺습니다.

```markdown
---

<div align="center">
  <sub>기술 골목의 평화와 자유는 우리가 지킨다 · <strong>고맙습니다. 냐-옹.</strong></sub>
</div>
```

운영 가이드(LEAK_CLEANUP, RECRUITMENT 등)는 변형해서 사용하셔도 됩니다.

- `누출은 사냥감입니다 · 고맙습니다. 냐-옹.`
- `작은 일을 가까이서 합니다 · 고맙습니다. 냐-옹.`

## 적용 우선순위

1. 모든 레포 README (외부 첫 인상이 되는 곳)
2. `manifesto/MANIFESTO.md`, `GANGNYANG.md` (자경단 정체성 문서)
3. 운영 가이드 (`LEAK_CLEANUP`, `RECRUITMENT`, `PILOT_SELECTION`)
4. 트랙별 핸드북 챕터 (작성 시점에)
